## BIOS(Basic Input Output System)

電源投入と同時に実行sれる。
ハードウェアを初期化し、補助記憶装置からブートローダーを呼び起こす。
またキーボードやディスプレイなどの入出力装置を,プログラムから利用するためのサービスを提供する
プログラムは、ハードウェアに直接アクセスするのではなく、これらBIOSを利用することでハードウェア差を気にしなくてよくなる。

近代的なOSは、この抽象化層を使わず、OSのデバイスドライバで直接制御する場合が多い。

1980-1990まで主流であったDOSはディスク、キーボード、画面表示など、BIOSのサービスに依存していた。
Windows Windows NT, Linux, プロテクトモードでの動作するOSは、一般的にOSのロードと起動までを行い、
起動後はBIOS割り込みルーチンを仕様しない。

## 起動
電源起動時、ハードウェアの初期化およびブートローダを読み込むプログラムが必要になる。
これらのプログラムを、IPL(Initial Program Loader)と呼ばれる。

プロセッサがリセットされると、リアルモードのアドレス空間1MBのうち、最後尾付近のアドレスにある命令を実行する。
そのアドレスに記されたジャンプ命令により、BIOSの実行コードにジャンプして処理が開始される。
電源が投入された場合(コールドブート)は、Power On Self Test (POST)が実行され、リセットボタンが押された場合(ウォームブート)は、POSTを省略し、メモリのテストにかかる時間を節約する。

Power On Self Testは、CPU, RAM, 割り込み、DMAコントローラなどの搭載デバイスの他に、チップセット、ビデオカード、キーボード、ハードディスク、光学ドライブ、などのハードウェアのシステム構成をテスト、認識、初期化する。

次いで、BIOSは起動デバイスに指定された記憶装置を探し、ブートローダプログラムを見つけてPCに実行するようロードする。
